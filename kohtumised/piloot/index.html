(põhimõtteliselt) kõik GITi operatsioonid on lokaalsed, kasutaja arvutis

working directory - staging area
modified - staged - commited
(ehk siis ...)
a) muudad faile
(sinu töökaust, working directory)
b) valid, millised failid lähevad järgmisse commiti
(git lisab need staging areasse)
c) teed commiti
(git muudab stage'itud failid snapshotiks)
ja see põhimõtteliselt ongi töövoog gitis
kuid mõned terminid veel
local - sinu lokaalne repo
remote - repo kuskil serveris
branch - repo üks haru / versioon (+ tegevus)
master - repo "vanemharu", luuakse alati automaatselt
checkout - haru "väljavõtmine" repost working diri
clone - remote repo koopia (+ tegevus)
fetch - tõmba remote'ist muudatused
merge - mesti muudatused (remote'ist või harust)
pull - tõmba remote'ist muudatused ja mesti ühe liigutusega
fork - koopia mõnest võõrast remote'ist (!= RW privileegid, olemas ainult R)
pull request - palun mesti minu muudatused, mul ei ole RW õiguseid sinu repole, ainult R

Eesmärk 1

*) seadista repo oma kohalikus masinas
*) lisa sellesse fail nimega README.md
# loo kaust

$ cd mingi/kaust/minu/masinas
$ mkdir minurepo
$ cd minurepo
                        
# initsialiseeri repo

$ git init
$ git status
                        
# lisa fail working directory'sse

$ echo test > README.md
                        
# kontrolli staatust

$ git status
                        
# stage'i muudatused

$ git add README.md
                        
# commiti muudatused

$ git commit -m "Repo seadistus"

Eesmärk 2

*) loo uus haru (branch)
*) muuda README.md ja salvesta muudatused loodud harusse
*) lisa uude harusse uus fail
*) mesti muudatused master harusse 
# navigeeri käsureal oma repo asukohta (kui vaja)

$ cd mingi/kaust/minu/masinas/minurepo
                        
# loo uus haru

$ git checkout -b katse-1
                        
# 1. ava README.md mõnes teksti-sõbralikus-editoris
#    (Notepad++, Atom, Notepad? vms "formattimisvaba")
                        
# 2. lisa sinna H1 stiiliga pealkiri markdownis
                        
# 3. lisa sinna vähemalt 2 lõiku tavalist teksti
#    (+ lisaks 1 hüperlink ja 1 pildifail)
                        
# 4. lisa sinna H2 stiiliga pealkiri
                        
# 5. lisa kõige lõppu tabel (3 veergu, 3 rida)
                        
# navigeeri käsureal oma repo asukohta (kui vaja)

$ cd mingi/kaust/minu/masinas/minurepo
                        
# lisa uus fail

$ echo test > uusfail.txt
                        
# kontrolli staatust

$ git status
                        
# stage'i muudatused (NB! -A switch lisab ootejärjekorda kõik failid)

$ git add -A
                        
# commiti muudatused

$ git commit -m "Markdowni kirjutamise testimine"
Ja võib-olla aja jooksul mingid muudatused veel
(kuid mestime nüüd muudatused master harusse tagasi) 
# checkout master haru

$ git checkout master
                        
# kontrolli staatust, midagi pole muutunud

$ git status
uuri File Exploreris oma töökausta

NB! need ei ole kadunud :)
(need muudatused on alles, kuid eelnevalt loodud katse-1 harus, kuid vaatame hetkel master haru) 
# loetle kohalikud harud

$ git branch
                        
# merge haru (katse-1) masterisse

$ git merge katse-1
                        
# kontrolli staatust

$ git status
uuri File Exploreris uuesti oma töökausta
(README.md on nüüd nähtav vana sisuga ja uus lisatud fail ka olemas)

Eesmärk 3

*) publitseeri oma repo, et see oleks teistele nähtav
# vali oma "teenusepakkuja" ja loo repo serveril (nt GitHubis)
http://github.com
või
http://bitbucket.org
või
http://gitlab.com
või
... 
# seejärel navigeeri käsureal oma repo asukohta (kui vaja)

$ cd mingi/kaust/sinu/masinas/minurepo
                        
# "ühenda" oma lokaalne (local) repo kaugrepoga (remote)
# NB! kontrolli URLi - sõltub valitud "teenusepakkujast"

$ git remote add origin https://github.com/<kasutajanimi>/<reponimi>.git
                        
# "lükka" lokaalse repo master haru kaugreposse

$ git push -u origin master
                        
# pärast mida võib minna veebibrauserisse oma tööd imetlema
